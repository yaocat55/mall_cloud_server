## 新增接口时检查

```markdown
- [ ] 这个接口需要权限控制吗？
      → 不需要 → 加入 PERMIT_ALL_URLS 白名单并注明理由
      → 需要 → 继续往下填

- [ ] 对应操作的权限标识（从本表找同类）: _______________

- [ ] auth_menu 表里有对应的记录吗？
      → 有 → 直接写 `@PreAuthorize("hasAuthority('xxx')")`
      → 无 → 等待 seed SQL 更新后再上线

- [ ] 需要数据权限过滤吗？
      → 否 → 普通 @PreAuthorize
      → 是 → 额外改 BaseService 数据权限逻辑
```
