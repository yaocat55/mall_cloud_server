# Admin 前端功能规划

## 核心理念

超级管理员能看到**全部菜单**，但唯一专属操作是**用户管理**（增删改查账号）。

角色、菜单、权限在架构设计阶段就已写死，不开放给前端配置：

```
用户创建 → 选部门（运营部/客服部/财务部）
         → 选岗位（主管/专员/...）
         → 权限自动关联（部门 + 岗位 → 角色 → 菜单 → 按钮权限）
```

## 超级管理员可见的全部页面

### 系统管理

```
用户管理 → /system/users
  创建/编辑/删除后台管理员账号。
  可设置的字段：用户名、手机号、邮箱、部门（下拉，自动从数据库读取）、岗位（下拉，自动从数据库读取）、状态（启用/禁用）。
  部门 + 岗位决定角色，角色决定权限——选了运营部+运营主管，自动获得运营角色的全部权限。
```

### 商品管理

```
商品列表 → /products
  商品的增删改查。商品名称、价格、库存、描述、上下架状态。
  运营日常维护的核心页面。

分类管理 → /products/categories
  商品分类树（多级树形结构）。新增/编辑/删除分类。
  支持按分类名称搜索筛选。
  比如：手机→苹果手机、服装→男装。

品牌管理 → /products/brands
  品牌列表。新增/编辑/删除品牌。
  比如：苹果、华为、耐克。

单位管理 → /products/units
  商品计量单位。新增/编辑/删除单位。
  比如：个、箱、KG、米。

属性管理 → /products/attributes
  商品规格属性。新增/编辑/删除属性及属性值。
  比如：颜色（红/蓝/黑）、尺寸（S/M/L）。

商品分组 → /products/groups
  商品分组管理，用于营销活动选品。

商品图片 → /products/photos
  上传、管理商品图片。支持分组归类。

首页管理 → /products/index
  管理 C 端首页展示内容：
  ├── 轮播图 → 首页顶部大图，新增/编辑/删除/排序
  ├── 公告管理 → 首页公告文字，新增/编辑/删除
  └── 推荐商品 → 首页推荐的商品列表，新增/编辑/删除/排序
```

### 订单管理

```
订单列表 → /orders
  查询所有用户订单。按订单号/用户/状态/时间范围筛选。
  查看订单详情（商品清单、金额、物流、收货地址）。
  客服日常处理售后的核心页面。

配送地址 → /orders/delivery
  订单关联的收货地址管理。查询/新增/编辑/删除。

退货审核 → /orders/returns
  用户发起的退货退款申请。
  客服审核：审批通过或拒绝（需填写拒绝原因）。
  查看退货详情（商品、凭证图片、退款金额）。
```

### 营销管理

```
优惠券管理 → /marketing/coupons
  优惠券的增删改查。设置优惠券类型（满减/折扣/现金）、面值、使用门槛、有效期。
  运营配置营销活动用。

秒杀管理 → /marketing/seckills
  秒杀活动的增删改查。设置秒杀商品、价格、库存、时间段。
  运营配置限时抢活用。

发放记录 → /marketing/provides
  优惠券发放记录列表。查询哪些用户收到了优惠券。

领取记录 → /marketing/receives
  用户主动领取优惠券的记录列表。
```

### 基础数据

```
图片库 → /basic/photos
  图片资源中心。上传图片、按分组浏览、删除。
  所有图片供商品/首页/公告等模块引用。

图片分组 → /basic/photo-groups
  图片分组的增删改查。运营对图片进行分类管理。

敏感词管理 → /basic/sensitive
  敏感词库的增删改查。内容发布时自动过滤。
  如：政治敏感词、广告词、色情词。

短信记录 → /basic/sms
# 短信发送记录列表。查看发送时间、手机号、内容、状态（成功/失败）。
  仅查询（只读），不提供新增和编辑。

通知管理 → /basic/notices
  站内通知的增删改查。支持全量推送或指定用户推送。
  客服发送系统通知用。

```

### 商品评价

```
评价管理 → /comments
  用户对商品的评价列表。运营可以回复评价、审核显示/隐藏。
  差评跟踪和回复是客服日常工作之一。
```

## 各角色可见页面

### 运营

```
商品管理 → 全部（商品/分类/品牌/单位/属性/分组/图片/首页）
营销管理 → 全部（优惠券/秒杀/发放/领取）
基础数据 → 图片库/图片分组/敏感词/通知
商品评价 → 评价管理
```

### 客服

```
订单管理 → 订单列表/配送地址/退货审核（全部操作权限）
商品评价 → 评价管理（回复、审核）
```

### 财务

```
订单管理 → 订单列表（仅查看金额和退款金额，不可修改订单状态）
```

## 不开发的前端页面

| 页面 | 原因 |
|:----|:------|
| 菜单管理 | 路由路径 + 权限标识由开发通过 DB 预设 |
| 角色管理 | 角色与部门的对应关系已写死 |
| 部门管理 | 部门为一次性配置，由开发维护 |
| 岗位管理 | 岗位为预设值，由开发维护 |
| 字典管理 | key-value 技术常量，运营不需要理解 |
| 定时任务 | 技术运维，非运营职责 |

## 技术实现

- admin-bff 提供 `POST /admin/v1/user/page`（聚合：用户列表 + 部门列表 + 岗位列表）
- admin-bff 提供 `GET /admin/v1/user/{id}/edit-data`（聚合：用户详情 + 部门树 + 岗位列表）
- 后端根据用户的 `dept_id` + `job_id` 自动确定 `role_id`，写入 `auth_user_role`
- 菜单树由后端根据 `auth_role_menu` 查询返回，前端只负责渲染
