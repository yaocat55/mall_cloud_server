## 阶段 0：止血（Week 1-2）

> 目标：**不会因为超卖、重复退款、库存混乱赔钱。** 这些不改不上线。

### Week 1 — 库存系统（Redis 预减 + 本地缓存 + MySQL 兜底）

**背景：** 当前 `reduceStockBatch` 直接扣 MySQL，无缓存无预减。高并发下 MySQL 扛不住，超时取消也不回滚，必赔钱。

**目标架构：**

```
扣库存链路：
下单 → Caffeine（200ms TTL）→ Redis DECR → MySQL 定时回写
                                  │
                            热商品分片键
                         stock:pid:{id}:{shard}
```

**按天排期：**

| 日 | 任务 | 涉及服务 | 产出 |
|---|------|---------|------|
| 1 | Redis 库存接口 + 数据初始化 | `mall-product` | `StockService` 接口；Redis `stock:pid:{id}` Key 设计；启动时从 MySQL 加载库存到 Redis |
| 1-2 | Caffeine 本地缓存接入 | `mall-product` | 热商品 TOP 100 自动识别；Caffeine `expireAfterWrite(200ms)`；穿透回源 Redis |
| 2-3 | 热商品分片键 | `mall-product` | `stock:pid:{id}:{shard}` 分片 DECR；`StockService.deductShard()` 接口；保底 1024 片 |
| 3 | 下单流程改走 Redis | `mall-order` | `OrderService` 下单改为调 Redis DECR；失败回滚 INCR 补回；超时/取消改为 INCR |
| 3-4 | 定时任务回写 MySQL | `mall-product` | 每分钟扫描 Redis 已售罄 Key；批量 Update MySQL `sold_quantity`；`product_stock_log` 流水表 |
| 4 | 下线 MQ 库存回滚 | `mall-order` | 删除 `STOCK_ROLLBACK_TOPIC` 生产者和消费者；兜底释放锁库存的定时任务统一由新架构完成 |
| 4-5 | 支付入库 | `mall-pay` | 新建 `pay_order` 表；支付成功写库；支付宝回调做幂等 |
| 5 | 联调验证 | 全链路 | 下单 → Redis DECR → 支付 → 定时回写 MySQL → 超时释放 → 库存对账一致 |

**产出物：**
- `StockService` 接口（普通扣减 + 分片扣减 + 本地缓存 + 回写 MySQL）
- 定时任务 `StockSyncToMysqlJob`（每分钟，扫 Redis Key 回写）
- `product_stock_log` 流水表（所有库存变动可追溯）
- `pay_order` 表（支付流水持久化）

## 库存系统架构速查

```
三层降级：
                                    MySQL 定时回写
         ┌──────────────────────────────────────┐
         │                                      │
  请求 → 本地缓存 → Redis DECR → 业务处理        │
         (Caffeine    (stock:pid)   (下单成功)   │
          200ms TTL)                (超时 INCR)  │
                                      │         │
                                   库存流水     └→ StockSyncJob
                                   (product_    (每分钟，清理已售罄
                                    stock_log)   Key + 回写 MySQL)

热商品分片：
  stock:pid:1001 → 普通商品，单 Key
  stock:pid:1001:0 ~ 1023 → 热商品，随机分片 DECR

兜底对账（每日凌晨）：
  SELECT SUM(sold_quantity) FROM product_product
  → 对比 Redis 中所有 Key 的 SUM + MySQL 剩余库存
  → 不一致 → 告警 + Redis 全量重载
```
