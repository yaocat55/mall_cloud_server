# 网关路由与白名单

## 路由表

| 路径前缀 | StripPrefix | 转发目标 | 说明 |
|----------|:----------:|----------|------|
| `/api/admin-api/**` | 2 | `lb://mall-admin-api` | 认证/权限服务直连 |
| `/api/admin/**` | 1 | `lb://mall-admin-bff` | 管理后台 BFF |
| `/api/mobile/**` | 1 | `lb://mall-mobile-bff` | 移动端 BFF |
| `/api/basic/**` | 2 | `lb://mall-basic-api` | 基础服务 |
| `/api/customer/**` | 2 | `lb://mall-customer-api` | C 端用户服务 |
| `/api/product/**` | 2 | `lb://mall-product-api` | 商品服务 |
| `/api/marketing/**` | 2 | `lb://mall-marketing-api` | 营销服务 |
| `/api/order/**` | 2 | `lb://mall-order-api` | 订单服务 |
| `/api/pay/**` | 2 | `lb://mall-pay-api` | 支付服务 |
| `/api/recommend/**` | 2 | `lb://mall-recommend-api` | 推荐服务 |
| `/api/message/ws**` | 2 | `lb:ws://mall-message-api` | WebSocket |
| `/api/message/**` | 2 | `lb://mall-message-api` | 消息服务 |

> BFF 路由使用 `StripPrefix=1`（只去掉 `/api`），因为 BFF 控制器本身带 `/admin` 或 `/mobile` 前缀。
> 其他服务使用 `StripPrefix=2`（去掉 `/api` + `/service-name`）。

## JWT 白名单

白名单配置在 Nacos `mall-gateway-dev.yaml` → `gateway.filter.noAuth` 中。

**匹配规则：**
- 末尾带 `/` 的是前缀匹配（如 `/api/basic/v1/commonSmsRecord/` 匹配其下所有路径）
- 末尾不带 `/` 的是精确匹配（如 `/api/admin/v1/auth/login` 只匹配路径本身）

### Admin BFF（公开接口，精确匹配）

```
/api/admin/v1/auth/login
/api/admin/v1/auth/loginByPhone
/api/admin/v1/auth/getCode
/api/admin/v1/auth/testLogin
```

### Mobile BFF（公开接口，精确匹配）

```
/api/mobile/v1/auth/login
/api/mobile/v1/auth/loginByPhone
/api/mobile/v1/auth/getCode
```

### 基础服务（前缀匹配，全开放）

```
/api/customer/v1/mobile/user/
/api/basic/v1/commonSmsRecord/
/api/basic/v1/image/
/api/basic/v1/file/
/api/basic/v1/commonSensitiveWord/
```

### 移动端商品/首页/支付（前缀匹配，全开放）

```
/api/product/v1/mobile/
/api/mobile/v1/home/
/api/mobile/v1/product/
/api/mobile/v1/coupon/
/api/mobile/v1/user/area
```

### 支付

```
/api/pay/v1/mobile/pay/
```

## JWT 鉴权流程

```
客户端请求
  │
  ├─ 无 token → 匹配白名单？→ 是 → 放行到下游
  │                              → 否 → 401 拒绝
  │
  └─ 有 token → JWT 验签（Keys.hmacShaKeyFor）
                → 验签通过？→ 是 → 透传 X-User-Id / X-User-Name / X-Roles → 放行
                            → 否 → 401 拒绝
```

下游服务通过 `AuthApiInterceptor`（处理 Header 或 JWT）和 `JwtTokenFilter`（admin 专属 Redis 黑名单）完成身份设入 SecurityContext。
