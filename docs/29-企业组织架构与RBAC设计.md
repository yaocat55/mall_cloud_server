# 企业组织架构与 RBAC 设计

## 部门架构

```
运营部   — 商品上架、营销活动、首页管理
客服部   — 订单查询、退货审核、消息通知
财务部   — 订单对账、退款审核
```

- 超级管理员不属于任何部门，拥有系统全部权限
- 运营部员工只能操作本部门关联的业务数据
- 客服部员工可查看全部订单，但仅操作售后相关功能
- 财务部员工可查看订单金额数据，但不能修改商品和订单状态

## 角色与权限

### 超级管理员

```
权限范围：全部 106 个菜单
数据范围：ALL（全部数据）
说明：系统配置、角色分配、用户管理。仅限 1-2 人持有。
```

### 运营

```
权限范围：
  ✅ 商品管理（商品/分类/品牌/属性/单位）
  ✅ 营销管理（优惠券/秒杀）
  ✅ 首页管理（轮播图/公告/推荐商品）
  ✅ 图片管理
  ✅ 敏感词管理
  ❌ 系统管理（用户/角色/菜单/部门/岗位）
  ❌ 订单管理
  ❌ 消息通知

数据范围：DEPT_ONLY（仅本部门数据）
说明：日常运营工作，不涉及系统配置和订单。
```

### 客服

```
权限范围：
  ✅ 订单管理（订单查询/配送地址/退货审核）
  ✅ 商品评价管理
  ✅ 消息通知
  ❌ 商品管理（不能修改商品信息）
  ❌ 营销管理
  ❌ 系统管理

数据范围：ALL（全部订单，售后不分部门）
说明：售后处理，需要查看所有用户的订单。
```

### 财务

```
权限范围：
  ✅ 订单管理（只读：查看订单金额/退款金额）
  ✅ 退货审核（退款相关）
  ❌ 商品管理
  ❌ 营销管理
  ❌ 系统管理

数据范围：ALL
说明：对账和退款审核，不涉及业务操作。
```

## 用户分配

| 用户 | 角色 | 部门 |
|:----|:----|:------|
| admin | 超级管理员 | — |
| （运营人员） | 运营 | 运营部 |
| （客服人员） | 客服 | 客服部 |
| （财务人员） | 财务 | 财务部 |

## 数据范围说明

| 范围 | 含义 | 适用角色 |
|:----|:----|:---------|
| ALL | 全部数据 | 超级管理员、客服、财务 |
| DEPT_ONLY | 仅本部门数据 | 运营 |
| SELF | 仅本人数据 | — |

## 技术实现

- 菜单权限：通过 `@PreAuthorize("hasPermission(...)")` 或前端路由控制
- 数据权限：通过 `DataScope` 注解在 MyBatis Mapper 层拼接 SQL 条件
- 权限标识：使用 `permission` 字段，格式如 `product:list`、`order:approve`
